# Community Activation Plan — CMO Deliverables

Plan and copy for activating the community around the public launch (issue #39). Pairs with
`docs/marketing/publish-ready/community-copy-pack.md` and `measurement-plan.md`.

## Week-1 activation calendar

| Day | Theme | Action |
|-----|-------|--------|
| 1 | Install/try path | Publish announcement + install/usage CTA across all channels. |
| 2 | Standards / CI gates | Post the branch-protection, CodeQL, Dependabot, secret-scanning story. |
| 3 | Offline / failover | Tell the local-first / keeps-working-offline narrative. |
| 4 | Contributor onboarding | Share the dev guide + Agent Runtime Contract; flag `good first issue`. |
| 5 | Week summary + next preview | Recap metrics, thank contributors, tease what's next. |

## Activation plays

### Newcomer welcome
- Pin the Discord welcome + the setup wizard steps.
- Auto-reply in issues: point to pistisai.app and the runtime + OS they're on.

### Failure-report loop
- Every failure report gets a triage label and an acknowledgement within 24h.
- Surface the top 3 recurring failures in the week-1 recap.

### Contributor funnel
- Label `good first issue` items and link the dev guide in the issue template.
- Push directly to `main` per AGENTS.md — keep the barrier low.
